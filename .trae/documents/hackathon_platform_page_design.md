# 页面设计说明（桌面优先）

## 0. 全局设计规范

### 0.1 Layout（全局）
- 桌面端以 12 栏 CSS Grid 为主（max-width: 1200–1280px，居中），页面内关键区域使用 Grid + Flex 混合。
- 顶部导航 Sticky（默认 72px 高度），二级锚点导航在官网页采用 Sticky Tabs（滚动高亮）。
- 断点：Desktop ≥ 1024；Tablet 768–1023；Mobile < 768（在本文仅作为适配原则，不做移动优先）。

### 0.2 Meta 信息（全局）
- Title 模板：`{活动名} | 开源黑客松`、后台页面为 `{功能} | 管理后台`。
- Description：一句话概述 + 关键节点（报名/提交/投票/公布）。
- Open Graph：og:title、og:description、og:image（活动海报）、og:type=website。

### 0.3 Global Styles（设计 Token）
- 背景：#FFFFFF；分区浅底：#F5F5F7；文字主色：#1D1D1F；次级文字：#6E6E73。
- 品牌强调色：#0071E3（按钮/链接/高亮）。
- 圆角：卡片 16px；按钮 12px；输入框 12px。
- 阴影：`0 8px 24px rgba(0,0,0,0.08)`（卡片悬浮）。
- Typography：
  - H1 40/48，H2 28/36，H3 20/28，Body 16/24，Caption 13/18。
- 交互状态：
  - Primary Button：默认 #0071E3；hover 加深 8%；disabled 40% 透明。
  - Link：默认 #0071E3；hover 下划线 + 轻微变深。
  - 表单校验：错误 #D70015；成功 #34C759。

## 1. 对外报名官网（活动官网页）

### 1.1 Layout
- 顶部：透明玻璃拟态导航（随滚动加深背景模糊）。
- 主体：分段式纵向堆叠（Stacked Sections），每段使用 2 列 Grid（内容/插图或信息卡）。
- 右侧（可选）：桌面端显示“关键节点卡片”（报名截止/提交截止/投票期/公布时间）。

### 1.2 Meta
- Title：`{活动名} | 对外报名官网`。
- Description：`{活动一句话主题}。报名截止 {date}，提交截止 {date}。`。

### 1.3 页面结构 & 区块
1) Top Nav
- 左侧 Logo + 活动名；右侧：`赛事说明 / 报名 / 日程 / 规则 / FAQ / 投票`。
- 右上角主按钮：`立即报名`（未登录跳登录）。

2) Banner（Hero）
- 大标题（活动名）+ 副标题（主题口号）+ 关键信息（时间/地点或线上）。
- CTA：Primary `立即报名`；Secondary `查看日程`。

3) 赛事说明
- 三张信息卡：面向人群、参赛形式、提交物（精简文案）。

4) 报名
- 状态条：`报名中/已截止/即将开始`。
- 两个入口：`选手报名`（强引导）、`评委报名/注册`（次入口，若保留）。
- FAQ Inline：报名需要什么/如何组队/是否收费。

5) 日程
- 时间轴组件（Timeline）：分节点显示（报名/开发/提交/评审/投票/公布）。

6) 规则
- Accordion（折叠面板）：参赛规则、作品要求、评审规则、投票规则。
- 每条规则支持“复制链接”（便于引用）。

7) FAQ
- 搜索框（仅客户端过滤）+ 分类 Tab（报名/组队/提交/投票/隐私与版权）。

8) Footer
- 主办方信息、联系邮箱、版权声明。

### 1.4 关键交互
- 锚点导航滚动高亮；点击平滑滚动。
- 报名 CTA：未登录 -> 登录页；登录后 -> 报名页。

## 2. 登录/注册页（四类角色统一入口）

### 2.1 Layout
- 居中单卡片（宽 420–480px），左侧可选插画（≥1200px 时双列）。

### 2.2 Meta
- Title：`登录 | 开源黑客松`。

### 2.3 组件与内容
- 角色选择（Segmented Control）：`管理员 / 评委 / 选手 / 大众投票`。
- 登录方式随角色切换：
  - 管理员：邮箱+密码（可加 2FA 提示占位）。
  - 评委：邮箱+密码（含“审核中提示”态）。
  - 选手：邮箱+密码 + GitHub 登录按钮。
  - 大众投票：邮箱验证码（输入邮箱 -> 发送验证码 -> 验证登录）。
- 通用：错误提示（顶部 Banner）、加载态（按钮 spinner）、隐私与条款链接。

## 3. 报名页（选手）

### 3.1 Layout
- 左侧步骤条（Steps）：`个人信息 -> 团队 -> 确认`；右侧表单。

### 3.2 结构
- 个人信息：昵称、技能标签（Tag 输入）、参赛经验（单选）。
- 团队：创建团队 / 加入团队（输入邀请码或选择已有邀请）。
- 确认：展示报名摘要 + 勾选同意规则。

### 3.3 状态
- 已报名：显示报名状态与团队信息，只允许在截止前编辑。

---

## 4. 评委工作台 / 在线评审页

### 4.1 Layout
- 工作台：左侧导航（分配列表/进度），右侧内容区。
- 评审页：左右分屏（左：作品信息与材料；右：评分表单与评语）。

### 4.2 核心组件
- 分配列表：表格/卡片切换；筛选（未评/已评）。
- 评分：四项维度（创新/技术/完成度/展示），滑块或数字输入。
- 评语：多行文本 + 字数提示。
- 提交：提交前二次确认（不可修改提示策略由产品决定）。

## 5. 大众投票页

### 5.1 Layout
- 顶部筛选栏（Sticky）：搜索、赛道/标签筛选、排序（热度/最新）。
- 作品列表：卡片 Grid（桌面 3 列）。

### 5.2 组件
- 作品卡：封面、标题、团队名、简述、投票数。
- 投票按钮：未登录触发登录；已投票显示 disabled + 提示剩余次数（如有）。
- 反刷提示：验证码弹窗/冷却倒计时提示位（如启用）。

## 6. 管理员后台（含 AI 配置）

### 6.1 Layout
- 左侧 Sidebar（固定 260px）：`官网内容 / 赛程Session / 分配评委 / 用户与角色 / 投票设置 / AI配置`。
- 右侧主内容：表格 + 详情抽屉（Drawer）组合。

### 6.2 AI 配置页面（OpenAI 兼容口子）
- 表单字段（必填）：
  - `baseUrl`（示例占位：`https://api.openai.com/v1`）
  - `apiKey`（Password input；仅显示是否已设置，不回显明文）
  - `model`（输入或下拉：例如 `gpt-4o-mini`）
- 操作：`保存`、`测试连通性`（按钮）
- 安全提示：`apiKey 仅服务端保存与使用；前端不持久化明文`。
- 状态：保存成功 toast；测试失败展示错误原因（网络/鉴权/模型不可用）。
